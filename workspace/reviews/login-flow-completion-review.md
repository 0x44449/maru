# login-flow-completion 설계 리뷰

## 결론

이전 리뷰에서 지적한 핵심 문제는 해결됐다. 현재 문서는 라우팅 가드 방식, 인증 상태 모델링, `/me` 실패 처리, 동시성 제어까지 포함하고 있어 구현 진행 가능한 수준이다.

## 확인된 개선 사항

- Root layout이 세그먼트 기반 가드로 정리되어 잘못된 화면 선마운트 위험이 줄었다.
- `checkUser()`의 stale 응답 무시 규칙이 추가되어 상태 경합 위험이 낮아졌다.
- `/api/v1/users/me`의 200/401/404/5xx/네트워크 오류 처리 기준이 문서화됐다.
- `userFetchError`와 `error` 상태를 분리해 서버 오류와 미등록 사용자를 같은 상태로 취급하지 않게 됐다.

## 남은 메모

- `status === "error"`를 `(onboarding)/user-register` 화면으로 보내서 재시도 UI를 제공하는 구조는 설계상 가능하다.
- 다만 화면 의미상으로는 "미등록 사용자 등록"과 "사용자 조회 실패"가 섞이는 구조이므로, 구현 시 화면 문구와 재시도 UX를 명확히 분리해야 혼동이 적다.
- 이 항목은 보완 권장 사항이지, 현재 설계를 막을 수준의 문제는 아니다.

## 승인 여부

승인 가능. 구현 진행해도 된다.
