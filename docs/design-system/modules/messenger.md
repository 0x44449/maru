# Messenger Design System

Atena Design System 기반의 메신저/채팅 기능 전용 규격.
개별 컴포넌트를 먼저 정의하고, 마지막 섹션에서 화면 조립 방법을 안내한다.

> 기반: [`../DESIGN_SYSTEM.md`](../DESIGN_SYSTEM.md) / [`../tokens.json`](../tokens.json)
> 프리뷰: [`messenger-preview.html`](./messenger-preview.html)
>
> **상위 규격:** 아바타, Group Avatar, Notification Badge는 `DESIGN_SYSTEM.md` 규격을 따른다.

## 5. Message Bubble (메시지 버블)

메신저의 핵심 컴포넌트. 텍스트, 이미지, 파일, 링크 등 다양한 메시지 타입을 담는다.

### 5-1. 기본 규격

| 속성 | 보낸 메시지 (sent) | 받은 메시지 (received) |
|------|-------------------|----------------------|
| 배경 | `primary.600` | `surfaceRaised` |
| 텍스트 색상 | `#ffffff` | `textPrimary` |
| 최대 너비 | 320px | 320px |
| Border Radius | 16px (균일) | 16px (균일) |
| 패딩 | 10px 14px | 10px 14px |
| Font Size | 15px | 15px |
| Line Height | 1.5 | 1.5 |
| 정렬 | 우측 | 좌측 |

받은 메시지는 추가로 `border` 1px 적용.

### 5-2. Border Radius 상세

카카오톡 스타일 — 꼬리 없이 모든 모서리 균일한 16px 라운드.

```
보낸 메시지 / 받은 메시지 동일:
  TL:16  TR:16
  BL:16  BR:16
```

### 5-3. 연속 메시지 규칙

같은 사람이 연속으로 보낸 메시지는 간격만 달라진다. Radius는 항상 동일(16px).

| 위치 | 하단 간격 |
|------|----------|
| 독립 (solo) | 8px |
| 첫 번째 (first) | 2px |
| 중간 (cont) | 2px |
| 마지막 (last) | 8px |

### 5-4. 시간 표시 (Timestamp)

| 속성 | 값 |
|------|-----|
| Font Size | 11px |
| 색상 | `textTertiary` |
| 위치 (sent) | 버블 **왼쪽**, 하단 정렬 |
| 위치 (received) | 버블 **오른쪽**, 하단 정렬 |
| 버블-시간 간격 | 4px |

- 연속 메시지에서는 **마지막 메시지에만** 시간 표시.
- 안 읽은 수는 시간 위에 표시 (세로 배치).

### 5-5. 텍스트 메시지

기본 버블 규격 그대로.

- 링크: 밑줄 + 탭 가능
- 보낸 메시지 링크 색상: `#ffffff` (밑줄)
- 받은 메시지 링크 색상: `primaryText` (밑줄)

### 5-6. 이미지 메시지

| 속성 | 값 |
|------|-----|
| 최대 너비 | 280px |
| 최대 높이 | 360px |
| 최소 너비 | 120px |
| Border Radius | 버블과 동일 (꼬리 포함) |
| 캡션 패딩 | 8px 12px (이미지 하단) |

- 비율 유지, 최대 크기 내에서 fit
- 로딩 중: skeleton (이미지 비율 유지)
- 탭 시: 전체 화면 이미지 뷰어

**다중 이미지:**

| 개수 | 레이아웃 | 이미지 간격 |
|------|---------|-----------|
| 1장 | 단독 | — |
| 2장 | 1x2 가로 | 2px |
| 3장 | 1 큰 이미지 + 2 세로 | 2px |
| 4장 | 2x2 그리드 | 2px |
| 5장 | 2 상단 + 3 하단 | 2px |
| 6장 | 3x2 그리드 | 2px |
| 7장 | 3 상단 + 4 하단 | 2px |
| 8장 | 4x2 그리드 | 2px |
| 9장 | 3x3 그리드 | 2px |

### 5-7. 파일 메시지

```
┌──────────────────────────────────┐
│ [📄 36px]  filename.pdf          │
│            2.4 MB                │
└──────────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 아이콘 | Lucide `file` 계열, 36px |
| 아이콘 색상 (sent) | `rgba(255,255,255,0.7)` |
| 아이콘 색상 (received) | `textTertiary` |
| 파일명 | 14px, medium (500), 1줄 말줄임 |
| 파일 크기 | 12px, `textTertiary` (sent: `rgba(255,255,255,0.6)`) |
| 패딩 | 12px 14px |
| 최대 너비 | 280px |
| 아이콘-텍스트 간격 | 12px |

### 5-8. 링크 프리뷰

URL이 포함된 메시지의 링크 프리뷰. 텍스트 버블과 링크 프리뷰 카드는 독립된 요소로, 텍스트 버블 아래에 별도 카드로 표시된다.

```
┌──────────────────────────────────┐
│ 메시지 텍스트 https://...         │  ← 텍스트 버블
└──────────────────────────────────┘
┌──────────────────────────────────┐
│ [OG Image - 높이 160px]          │  ← 링크 프리뷰 카드 (독립)
│ 사이트명 · 12px                   │
│ 제목 · 14px, semibold            │
│ 설명 · 13px, 2줄 말줄임           │
└──────────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 카드 너비 | 280px (고정) |
| Border Radius | 16px |
| 배경 | `surfaceRaised` |
| 보더 | `border` 1px |
| OG 이미지 높이 | 160px (비율 유지, cover) |
| 텍스트 패딩 | 10px 14px |
| 사이트명 | 12px, `textTertiary` |
| 제목 | 14px, semibold (600) |
| 설명 | 13px, `textSecondary`, 최대 2줄 |

### 5-9. 텍스트+첨부 조합 메시지

텍스트와 이미지 또는 파일이 함께 포함된 메시지. 텍스트 버블 위, 첨부(이미지/파일) 아래로 각각 독립된 요소로 표시된다.

```
┌──────────────────────────────────┐
│ 텍스트 메시지 내용                 │  ← 텍스트 버블
└──────────────────────────────────┘
┌──────────────────────────────────┐
│ [이미지 또는 파일]                 │  ← 첨부 요소 (독립)
└──────────────────────────────────┘
```

- 텍스트 버블과 첨부 요소는 독립된 요소로, 각각의 규격(5-5~5-7)을 따른다.

## 6. Composed Message (조합 메시지)

아바타, 발신자 이름, 버블을 조합한 메시지 단위 컴포넌트.

```
[Avatar 36px] 발신자이름
              ┌──────────────────────┐
              │ 메시지 내용            │
              └──────────────────────┘
                              12:34
```

| 속성 | 값 |
|------|-----|
| 아바타 크기 | 36px (`avatar.md`) |
| 아바타 Border Radius | 12px (둥근 사각형) |
| 아바타-버블 간격 | 8px |
| 발신자 이름 | 13px, semibold (600) |
| 이름-버블 간격 | 4px |
| 버블-메타 간격 | 4px |

- 보낸 메시지(sent): 아바타/이름 없이, 우측 정렬
- 받은 메시지(received): 아바타 + 이름 + 버블, 좌측 정렬
- 연속 메시지에서는 첫 메시지에만 아바타/이름 표시, 이후 36px 빈 공간 유지

### 전송 상태

버블 옆에 표시. 보낸 메시지에만 적용.

| 상태 | 표시 내용 | 비고 |
|------|---------|------|
| 전송 중 | `loader` 아이콘 (12px, `textTertiary`) | 시간 미표시 |
| 전송 완료 (모두 읽음) | 시간만 표시 | 아이콘 없음 |
| 안 읽은 사람 있음 | 안 읽은 수 + 시간 | 숫자: `primary.600` (dark: `primary.300`), 11px, bold |
| 전송 실패 | `alert-circle` 아이콘 (14px, `error`) | 시간 미표시, 버블 opacity 0.6 |

- 안 읽은 수는 시간 **위**에 표시 (세로 정렬)
- 1:1 대화에서 안 읽으면 `1` 표시, 읽으면 숫자 사라짐
- 그룹 대화에서는 읽지 않은 인원 수 표시
- 전송 실패 시 아이콘 탭으로 재전송

## 7. Message Reply (답장 인용)

답장 메시지는 분할 버블 형태로, 상단에 원본 인용(reply-origin), 하단에 답장 본문(reply-body)이 하나의 버블 안에 배치된다.

```
┌──────────────────────────────────┐
│ 발신자이름                        │  ← reply-origin (상단)
│ 원본 메시지 내용 (1줄 말줄임)      │
├──────────────────────────────────┤
│ 답장 메시지 내용                   │  ← reply-body (하단)
└──────────────────────────────────┘
```

**reply-origin (상단 인용 영역):**

| 속성 | 값 |
|------|-----|
| 패딩 | 10px 14px |
| Font Size | 13px |
| Line Height | 1.4 |
| 배경 (sent) | `rgba(0,0,0,0.12)` |
| 색상 (sent) | `rgba(255,255,255,0.75)` |
| 배경 (received) | `rgba(0,0,0,0.03)` |
| 색상 (received) | `textSecondary` |

**reply-origin 발신자 이름 (rp-name):**

| 속성 | 값 |
|------|-----|
| Font Size | 12px |
| Font Weight | semibold (600) |
| 색상 (sent) | `primary.200` |
| 색상 (received) | `primary.600` |

**reply-body (하단 답장 영역):**

| 속성 | 값 |
|------|-----|
| 패딩 | 10px 14px |
| Font Size | 15px |
| Line Height | 1.5 |

인용 블록 탭 시 원본 메시지로 스크롤 이동 + `primaryBg` 배경 하이라이트 (0.5초 fade out).

## 9. Message Reaction (리액션)

메시지 버블 하단에 이모지 반응 칩을 표시.

```
메시지 버블
 [👍 3] [❤️ 1] [+]
```

| 속성 | 값 |
|------|-----|
| 칩 높이 | 28px |
| 칩 패딩 | 4px 8px |
| Font Size | 13px |
| Border Radius | `full` |
| 배경 (기본) | `surface` + `border` 1px |
| 배경 (내가 누른 것) | `primaryBg` + `primary` 보더 |
| 칩 간 간격 | 4px |
| 정렬 | 좌측(received) / 우측(sent) |
| 버블-리액션 간격 | -4px (살짝 겹침) |

- 추가 방법: 롱프레스(모바일) 또는 호버 시 [+] 버튼 → 이모지 피커

## 11. Message Input (메시지 입력)

카카오톡 스타일의 메시지 입력 영역. 입력 필드는 독립된 라운드 컨테이너이며, 버튼들은 입력 필드 외부에 배치된다.

```
[＋] ┌─────────────────────────────┐ [➤] [😊] [#]
     │ 메시지를 입력하세요...        │
     └─────────────────────────────┘
```

**레이아웃:** `[+] [input field] [send] [emoji] [#]`

| 속성 | 값 |
|------|-----|
| 입력 필드 Border Radius | 20px |
| 입력 필드 배경 | `surfaceRaised` |
| 입력 필드 보더 | `border` 1px |
| textarea | `field-sizing: content`, max-height 120px, 자동 확장 |
| Font Size | 15px |
| Line Height | 1.5 |

| 요소 | 스타일 |
|------|--------|
| 첨부 버튼 [＋] | 입력 필드 외부 좌측 |
| 전송 버튼 [➤] | 36px 원형 |
| 전송 버튼 (미입력) | `border` 배경 + opacity 0.5 |
| 전송 버튼 (입력됨) | `primary.600` 배경 + 흰색 아이콘 |
| 이모지 버튼 [😊] | 입력 필드 외부 우측 |
| 해시 버튼 [#] | 입력 필드 외부 우측 |

### 답장 모드

답장 시 입력 영역 상단에 reply-chip 표시. 입력 필드 외부에 위치.

```
┌─────────────────────────────────────────────────┐
│ [|] 발신자이름                               [✕] │  ← reply-chip
│ [|] 원본 메시지 내용 (1줄 말줄임)                 │
├─────────────────────────────────────────────────┤
│ [＋] [입력 필드]                    [➤] [😊] [#] │
└─────────────────────────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 패딩 | 8px 12px |
| 배경 | `surface` |
| 보더 | `border` 1px |
| Border Radius | 12px |
| 인디케이터 바 | 2px × 28px, `primary.500`, 좌측 |
| 발신자 이름 | 12px, semibold (600), `primaryText` |
| 메시지 텍스트 | 13px, `textSecondary`, 1줄 말줄임 |
| 닫기 버튼 | `x` 아이콘, `textTertiary` |

## 13. Chat Header (채팅방 헤더)

카카오톡 스타일의 대화방 상단 헤더.

```
┌─────────────────────────────────┐
│ [◀]  채팅방 이름  (3)   [🔍] [≡] │
│  44   gap8  flex1        44  44  │
└─────────────────────────────────┘
         높이: 56px
```

| 속성 | 값 |
|------|-----|
| 높이 | 56px |
| 배경 | `background` |
| 하단 보더 | `border` 1px |
| 좌우 패딩 | 4px |
| z-index | `sticky` (1100) |
| position | sticky, top: 0 |

### 내부 요소

| 요소 | 규격 |
|------|------|
| 뒤로가기 버튼 | 44×44px 터치 영역, 아이콘 24px (`chevron-left`) |
| 채팅방 이름 | 17px, semibold (600), `textPrimary`, 1줄 말줄임 |
| 참여자 수 | 13px, regular (400), `textTertiary`, 이름 우측 4px 간격 |
| 우측 버튼들 | 각 44×44px 터치 영역, 아이콘 24px |
| 아이콘 색상 | `textSecondary` |

**우측 버튼 구성:** `[검색 search]` `[메뉴 menu]`

### 배리언트

| 배리언트 | 설명 |
|----------|------|
| 그룹 채팅 | 채팅방 이름 + 참여자 수 표시 |
| 1:1 채팅 | 상대방 이름만 표시, 참여자 수 미표시 |
| 긴 제목 | `flex: 1` + `overflow: hidden` + `text-overflow: ellipsis` |
| 검색 모드 | 헤더가 검색바로 전환 |

- 채팅방 이름 + 참여자 수는 중앙이 아닌 **좌측 정렬** (뒤로가기 버튼 우측)
- 이름 영역은 `flex: 1`로 남은 공간 전체 차지

### 검색 모드

검색 버튼 탭 시 Header가 검색 입력 UI로 전환된다.

```
┌─────────────────────────────────┐
│  [검색 입력 필드         ] [취소] │
└─────────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 높이 | 56px (기본 Header와 동일) |
| 좌우 패딩 | 좌 16px, 우 12px |
| 입력 필드 | `flex: 1`, 14px, `surfaceRaised` 배경, `border` 1px, border-radius 8px, 높이 36px |
| 입력 필드 패딩 | 0 12px |
| 입력 필드 내부 gap | 8px |
| placeholder | "메시지 검색", `textTertiary` |
| 검색 아이콘 | 입력 필드 좌측 내부, `search` 16px, `textTertiary` |
| 클리어 버튼 | 입력 필드 우측 내부, `x-circle` 16px, `textTertiary`, 텍스트 입력 시에만 표시, 탭 시 입력 전체 삭제 |
| 취소 버튼 | 14px, medium (500), `primaryText`, 패딩 0 12px |
| 자동 포커스 | 검색 모드 진입 시 입력 필드에 포커스 |

### 검색 결과 네비게이션 바

검색 결과가 있을 때 검색 헤더 하단에 표시되는 이동 바.

```
┌─────────────────────────────────┐
│  3개의 결과 중 1번째     [▲] [▼] │
└─────────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 높이 | 44px |
| 배경 | `background` |
| 하단 보더 | `border` 1px |
| 좌우 패딩 | 16px |
| 결과 텍스트 | 13px, `textTertiary` |
| 이동 버튼 | 32×32px, border-radius 8px, `border` 1px, 아이콘 `chevron-up`/`chevron-down` 16px, `textSecondary` |
| 버튼 간격 | 4px |

**검색 결과 동작:**

| 동작 | 설명 |
|------|------|
| 검색 입력 시 | Message Area에서 일치하는 메시지 하이라이트 |
| 하이라이트 | 일치 텍스트에 `primaryBg` 배경 |
| 이동 | 네비게이션 바의 상/하 버튼으로 결과 간 이동 |
| 결과 없음 | Message Area 중앙에 "검색 결과 없음" 텍스트 |

## 14. Scroll to Bottom (하단 스크롤 버튼)

스크롤을 일정 이상 올렸을 때 Message Area 우하단에 표시되는 원형 버튼.

| 속성 | 값 |
|------|-----|
| 크기 | 40px 원형 |
| 위치 | Message Area 우하단, 우측 16px, 하단 16px |
| 배경 | `surfaceRaised` |
| 보더 | `border` 1px |
| 그림자 | `shadow.md` |
| 아이콘 | `chevron-down` 20px, `textSecondary` |
| Border Radius | `full` |
| 탭 동작 | 최하단으로 스크롤 |

### 읽지 않은 메시지 뱃지

| 속성 | 값 |
|------|-----|
| 위치 | 버튼 우상단, top -8px, right -4px |
| 규격 | 2번 Notification Badge와 동일 |

### 배리언트

| 배리언트 | 설명 |
|----------|------|

## 15. New Message Float (새 메시지 플로팅)

사용자가 스크롤을 올린 상태에서 새 메시지가 도착하면 Message Area 하단에 표시. 발신자 아바타 + 이름 + 메시지 미리보기를 한 줄로 포함.

```
┌──────────────────────────────────────┐
│ [Avatar 28px] 발신자이름 메시지... [↓] │
└──────────────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 위치 | Message Area 하단, 좌우 16px 마진 |
| 높이 | 44px |
| 배경 | `surfaceRaised` |
| 보더 | `border` 1px |
| Border Radius | 12px |
| 패딩 | 0 12px |
| 내부 gap | 8px |
| 그림자 | `shadow.md` |
| 아바타 | 28px (`avatar.sm`), 10px radius |
| 텍스트 레이아웃 | 이름 + 메시지 한 줄, baseline 정렬, gap 4px |
| 발신자 이름 | 13px, semibold (600), `textPrimary`, 축소 불가 |
| 메시지 미리보기 | 13px, `textSecondary`, 1줄 말줄임 |
| 화살표 아이콘 | `chevron-down` 16px, `textTertiary` |
| 탭 동작 | 최하단으로 스크롤 + 알림 사라짐 |

> Scroll to Bottom 버튼(14번)과 동시 표시되지 않는다. 새 메시지 도착 시 이 플로팅이 우선.

# Part 2. 화면 조립

위 컴포넌트들을 조합하여 채팅방 화면을 구성하는 방법.

---

## 전체 구조

```
┌─────────────────────────────────┐
│ ◀  채팅방 이름          🔍  ≡  │ ← 13. Header
├─────────────────────────────────┤
│                                 │
│         Message Area            │ ← Message Area
│         (flex: 1)               │
│                                 │
│  [Scroll ↓]  or  [새 메시지]    │ ← 14/15. 플로팅 요소
├─────────────────────────────────┤
│ [＋] [입력 필드]     [➤] [😊] [#] │ ← 11. Input Area
├─────────────────────────────────┤
│         Safe Area               │ ← Safe Area
└─────────────────────────────────┘
```

전체 레이아웃은 `100dvh` 높이의 세로 flex 컨테이너.

| 영역 | 높이 | 비고 |
|------|------|------|
| Header | 56px (고정) | `position: sticky`, `z-index: sticky (1100)` |
| Message Area | `flex: 1` | `overflow-y: auto` |
| Input Area | auto | 콘텐츠 높이에 따라 가변 (최대 textarea 120px) |
| Safe Area | `env(safe-area-inset-bottom)` | iOS 하단 여백 |

### Message Area

| 속성 | 값 |
|------|-----|
| flex | 1 (남은 공간 전체) |
| overflow-y | auto |
| overflow-x | hidden |
| 배경 | `background` |
| 좌우 패딩 | 16px |
| 상단 패딩 | 16px |
| 하단 패딩 | 8px |

### Message Area 내부 조합 순서

```
[3. Date Divider]
[6. Composed Message]
  └ 아바타 + 발신자 이름 (received만)
  └ [7. Reply] (있으면 분할 버블)
  └ 메시지 콘텐츠 (5-5~5-9)
  └ [8. Mention] (텍스트 내부 인라인)
전송 상태 + 시간 (버블 하단 외부)
[9. Reaction] (버블 하단, -4px 겹침)
[4. System Message]
[10. Typing Indicator] (최하단)
```

### Input Area 레이아웃

| 속성 | 값 |
|------|-----|
| 배경 | `background` |
| 상단 보더 | `border` 1px |
| 좌우 패딩 | 8px |
| 상하 패딩 | 8px |
| 세로 정렬 | 하단 정렬 (버튼들은 입력 필드 하단에 맞춤) |

#### 답장 모드

답장 시 Input Area 상단에 reply-chip이 추가된다.

```
┌─────────────────────────────────┐
│ 답장: 이름 — 메시지 내용...  [✕] │ ← reply-chip
├─────────────────────────────────┤
│ [＋] [입력 필드]     [➤] [😊] [#] │
└─────────────────────────────────┘
```

| 요소 | 규격 |
|------|------|
| reply-chip 높이 | 40px |
| reply-chip 좌우 패딩 | 12px |
| reply-chip 배경 | `surface` |
| reply-chip 하단 보더 | `border` 1px |
| 닫기 버튼 | `x` 아이콘 16px, 터치 영역 20px, `textTertiary` |

### 스크롤 동작

| 동작 | 설명 |
|------|------|
| 초기 진입 | 최하단(최신 메시지)으로 스크롤 |
| 새 메시지 수신 (하단 위치) | 자동으로 최하단 스크롤 |
| 새 메시지 수신 (스크롤 올린 상태) | 스크롤 유지, New Message Float(15번) 표시 |
| 과거 메시지 로드 | 상단 스크롤 시 로드, 스크롤 위치 유지 |

### 키보드 상호작용

| 동작 | 설명 |
|------|------|
| 키보드 등장 | Message Area 축소, Input Area가 키보드 위로 이동 |
| 스크롤 위치 | 키보드 등장 전 최하단이었으면 최하단 유지 |
| Safe Area | 키보드 활성 시 `safe-area-inset-bottom` 대신 키보드 높이 적용 |
| 키보드 닫기 | Message Area 원래 크기로 복원 |
| 입력 필드 탭 | 키보드 등장 + 자동 스크롤 |

### Safe Area

iOS 등 노치/홈 인디케이터가 있는 기기 대응.

| 적용 위치 | 설명 |
|-----------|------|
| 최외곽 컨테이너 | `min-height: 100dvh` |
| Input Area 하단 | `env(safe-area-inset-bottom)` 추가 패딩 |
| Header 상단 | `env(safe-area-inset-top)` (standalone 모드 시) |
